Introdução

Faala dev!
Seja muito bem-vindo ou bem-vinda ao quarto desafio da trilha de React Native do Ignite.
Nesse desafio você vai reforçar os conceitos de Push Notifications e Deep Linking no React Native.

Sobre o desafio

Nesse desafio, você vai adicionar as funcionalidades de Push Notification e Deep Linking ao projeto 03 da trilha de React Native: Ignite Gym. Para isso, vocês precisarão aplicar o que foi visto em aula:
Push Notification
OneSignal
Expo Dev Client
FCM e/ou APNS
Deep Linking (React Navigation)
Na aplicação Ignite Gym que foi desenvolvida por completa na trilha React Native.

Desenvolvendo o projeto

Para desenvolver esse projeto, recomendamos utilizar o OneSignal para Push Notifications, React Navigation para o Deep Linking e o Expo Dev Client como fluxo de trabalho no Expo.
Nas seções abaixos daremos mais informações sobre cada um desses três pontos principais do projeto.

Push Notifications

Para a correta implementação dessa funcionalidade com as tecnologias que recomendamos/usamos em aula, é necessário:
Criar uma conta e um projeto no OneSignal. Caso tenha dúvidas, pode assistir às aulas desse cluster.
(Android) Criar uma conta e um projeto no Firebase e ativar o FCM v1 (legacy). Caso tenha dúvidas, pode assistir às aulas desse cluster.
(iOS) Criar uma conta de desenvolvedor na Apple, assinar um dos planos e gerar o certificado p12. Caso tenha dúvidas, pode assistir às aulas desse cluster.
Com essa funcionalidade configurada, você deve implementar pelo menos 3 Push Notifications com conteúdos e públicos diferentes. Sinta-se a vontade para escolher os públicos e conteúdos que preferir. Deixaremos abaixo algumas sugestões:
Notificar usuário logado que o último treino dele foi há x dias e motivá-lo a voltar.
Notificar usuários de novidades no app (ex.: novos exercícios adicionados).
Notificar usuário logado ao final da semana sobre a quantidade de exercícios feitos e desafiá-lo a manter ou superar.
Se ainda não estiver logado, convidá-lo a entrar ou se cadastrar.
Por fim, caso queira se aprofundar/relembrar os conceitos da aula, deixaremos links de duas documentações relacionadas ao tema:
Expo SDK Setup
Client SDK Reference

Deep Linking

Para a correta implementação dessa funcionalidade com as tecnologias que recomendamos/usamos em aula, é necessário:
Utilizar o React Navigation para o gerenciamento dos linkings.
Utilizar o pacote uri-scheme para adicionar os schemes nativos no projeto.
Caso esteja com dificuldades nessa etapa, sugerimos que assista às aulas desse cluster.
Com essa funcionalidade configurada, você deve implementar:
Pelo menos 3 deep links.
Pelo menos 1 deep link com parâmetro.
Deep Links tanto para o fluxo autenticado quanto o não autenticado.
Tratamento para rota não encontrada (404). Para isso, sugerimos a leitura dessa documentação.
Por fim, caso queira se aprofundar/relembrar os conceitos da aula, deixaremos links de duas documentações relacionadas ao tema:
Deep Linking
Configuring Links

Expo Dev Client

A fim de permitir que você utilize o OneSignal e React Navigation sem grandes dificuldades para a implementação das Push Notifications e Deep Linking, recomedamos utilizar o Expo Dev Client visto que o Expo Go (Managed Workflow) não possui suporte.
Caso tenha dificuldade em configurar essa etapa, você pode assistir às aulas desse cluster.
Por fim, caso queira se aprofundar/relembrar os conceitos da aula, deixaremos link da documentação relacionada ao tema:
